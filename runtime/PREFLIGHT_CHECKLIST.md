# PREFLIGHT_CHECKLIST

Before any non-trivial action:

- [ ] What is the route class?
- [ ] Is this direct answer or action path?
- [ ] Is there a side effect?
- [ ] Does the side effect require approval?
- [ ] What evidence handle will prove the outcome?
- [ ] What is the verification method?
- [ ] What wording is allowed if verification fails?
- [ ] Is any memory/vault note being treated as truth without lifecycle authority?
- [ ] Is delegate/tool output being treated only as evidence?
- [ ] Will MainBrain-style synthesis speak last?
